# User Controller API Documentation

## Overview

The `UserController` is responsible for handling user-related
operations: 
- user registration
- user authentication

Authentication is implemented using JSON Web Token.

------------------------------------------------------------------------

## Base URL

/api/v1

------------------------------------------------------------------------

## Authentication Flow

1.  A user registers using the `/users` endpoint
2.  The user logs in using the `/sessions` endpoint
3.  The system returns a JSON Web Token
4.  The token must be included in all subsequent requests using the
    Authorization header

Authorization: Bearer `<token>`

------------------------------------------------------------------------

## Data Models

### UserDTO

{ "userName": "string", "password": "string", "role": "string" }

Fields:

- userName -- the unique username of the user
- password -- the user password
- role -- the role assigned to the user (for example: USER or ADMIN)
------------------------------------------------------------------------

## JSON Web Token Structure

The token generated by the application contains the following important
fields:

-   subject -- represents the username of the authenticated user\
-   issued at -- the time when the token was created\
-   expiration -- the time when the token expires (24 hours after
    creation)

------------------------------------------------------------------------

## Endpoints

### Register User

POST /api/v1/users

Creates a new user.

Request Body: \
{\
"userName": "john_doe",\
"password": "password123",\
"role": "USER"\
}

Response: {\
"id": "uuid",\
"userName": "john_doe",\
"role": "USER"\
}

Status Codes: 
- 200 OK -- user successfully created
- 400 Bad Request -- invalid input data

------------------------------------------------------------------------

### Login User

POST /api/v1/sessions

Authenticates a user and returns a JSON Web Token.

Request Body: {\
"userName": "john_doe",\
"password": "password123"\
}

Response: "json-web-token-string"

Status Codes: - 200 OK -- authentication successful\
- 401 Unauthorized -- invalid credentials

------------------------------------------------------------------------

## Error Handling

When authentication fails: { "error": "Invalid credentials" }

Errors are handled using the default behavior of the Spring framework

------------------------------------------------------------------------

## Security Notes

-   The JSON Web Token must be included in the request header:
    Authorization: Bearer `<token>`

-   The token expires after 24 hours

-   The secret key is currently hardcoded and should be moved to
    configuration

------------------------------------------------------------------------

## Recommendations

-   Use input validation\
-   Avoid returning entities directly\
-   Hash passwords securely\
-   Use correct HTTP status codes

------------------------------------------------------------------------

## Conclusion

This application programming interface is simple, clear, and suitable
for demonstration purposes.
