1. Quartz job să ruleze la un timp predefinit setat prin cron in application properties.
2. ⁠Jobul trebuie să scaneze folderul și să uploadeze fișierele dar să logheze explicit ce a făcut. Spre ex: Uploaded successful for file xxx. / Error for file xxx. ( in caz de eroare fișierele rămân pe loc și reîncărcăm la următorul run )
3. ⁠după upload cu succes fișierul este mutat intr un backup folder. Structura tine cont de data măcar: backup/20260124/xxx
4. ⁠calea de fișier de backup este configurabilă în application settings. La fel și formatul datei din sub folderul în care se salvează fișierele
5. ⁠deduplicare fisiere. Gen pentru fiecare fișier calculăm hashul și salvăm în baza nume + acest hash. La fiecare attempt de upload pt un fișier trebuie să verificăm dacă nu cumva este duplicat fișierul. Gen luăm din bază după nume acel hash, calculăm hashul fișierului de pe disk și dacă e egal atunci este același fișier. Dacă e duplicat îl mutăm în backup folder direct și menționăm în log asta.
6. ⁠un fișier cu eroare trebuie să fie reîncărcat de X ori. Apoi fișierul trebuie mutat intr un folder special din backup spre ex backup-errors/…
7. ⁠calea pt backup-errors sa fie configurabilă iar formatul pt errors este diferit: practic tu ai fișierul demo.csv. Dacă el nu poate fi uploadat de cele X ori de retry atunci îl mutăm în backup-errors/ cu numele 20260124-demo.csv . Practic vrem toate fișierele în același folder să fie ușor de investigat.
